# 10 — Post-Demo Notes

> *"A demo is not finished until feedback becomes decisions and tasks."*

---

# Post-Demo Capture Template

```text
Demo date:
Audience:
Presenter:
Environment:
Commit/version:
Demo result: pass/fail/partial
```

---

# Feedback Categories

```text
product value
UX clarity
AI quality
security/trust
missing features
technical concerns
demo issues
roadmap suggestions
```

---

# Questions to Capture

```text
Did the workflow feel valuable?
Was the MVP scope clear?
Was customer context enough?
Was the AI draft helpful?
Was human review clear?
Were role restrictions clear?
Any concern about data exposure?
Any blocker before repo skeleton/coding?
```

---

# Decision Log

Record:

```text
accepted changes
rejected changes
deferred features
new risks
new blockers
required doc updates
```

---

# Action Item Format

```text
Owner:
Task:
Priority:
Due:
Linked doc/task:
```

---

# Post-Demo Checklist

- [ ] Capture whether agent workflow felt clear.
- [ ] Capture whether viewer read-only behavior was understood.
- [ ] Capture any concern about mock auth or simulated providers.
- [ ] Confirm nobody interpreted AI draft as autonomous send.
- [ ] Document any mismatch between demo claims and implemented MVP.
- [ ] Convert approved follow-up work into backlog or doc updates.

---

# Rule

```text
Do not turn demo feedback directly into code before deciding whether it changes PRD, TDD, UX, API, DB, Security, or Backlog.
```
