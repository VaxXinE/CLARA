# CLARA P12 Beta Support SLA Policy

P12-PR-01 is complete. P12-PR-02 is complete. P12-PR-03 is complete. P12-PR-04 is current.

CLARA is not GA yet.

The beta feedback workflow is controlled and privacy-safe.

Feedback/support must not collect raw sensitive data.

No external support tool integration happens in this PR.

No auto-send or external ticket creation happens in this PR.

No provider/payment/AI/outbound activation happens in this PR.

## Support Severity

| Severity | Meaning | Beta response target |
| --- | --- | --- |
| S0 security/data exposure/blocker | Security issue, data exposure, auth/workspace failure, or beta stop-ship. | same day |
| S1 beta-breaking core workflow | Core beta workflow unusable with no safe workaround. | next business day |
| S2 important but workaround exists | Important issue with documented workaround. | 2 business days |
| S3 minor UX/docs issue | Minor UI, copy, docs, or low-risk confusion. | weekly review |
| S4 feedback/enhancement | Product feedback or enhancement idea. | backlog review |

Beta SLA is a support target, not a paid support guarantee.
