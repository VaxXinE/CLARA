# Book I — The Foundation Changelog

All meaningful changes to Book I should be documented in this file.

Book I is a constitutional document. Changes should be intentional, reviewed, and versioned.

---

## Versioning Policy

Book I follows semantic versioning.

### MAJOR Version

Used when foundational principles change.

Examples:

* Changing the mission.
* Replacing core values.
* Redefining AI philosophy.
* Changing constitutional authority.
* Removing or rewriting major principles.

Example:

```text
1.0.0 → 2.0.0
```

### MINOR Version

Used when new sections, clarifications, supporting principles, or additional chapters are added without changing the core meaning.

Examples:

* Adding a new supporting principle.
* Clarifying governance.
* Expanding glossary terms.
* Adding supporting appendix files.

Example:

```text
1.0.0 → 1.1.0
```

### PATCH Version

Used for minor corrections that do not change meaning.

Examples:

* Typo fixes.
* Formatting improvements.
* Broken link fixes.
* Minor wording improvements.
* Table formatting corrections.

Example:

```text
1.0.0 → 1.0.1
```

---

## Change Review Rules

Changes to Book I should follow these rules:

1. Every meaningful change must be reviewed.
2. Changes to principles require strong justification.
3. Trade-offs should be documented.
4. References should be updated when file names or chapter structure changes.
5. Version number should be updated when appropriate.
6. The changelog must be updated in the same pull request.
