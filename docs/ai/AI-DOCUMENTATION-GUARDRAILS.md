# AI Documentation Guardrails

## Required For AI Docs

AI documentation must define:

- Capability purpose.
- Context boundary.
- Data access boundary.
- Tool permissions.
- Memory behavior.
- Prompt ownership.
- Evaluation strategy.
- Safety guardrails.
- Privacy controls.
- Observability.

## Forbidden

Do not document AI as having:

- Unrestricted data access.
- Unrestricted tool execution.
- Unreviewed memory persistence.
- Permission bypass.
- Direct secret access.
