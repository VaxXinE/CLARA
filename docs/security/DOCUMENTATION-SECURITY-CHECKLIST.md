# Documentation Security Checklist

- [ ] No API keys.
- [ ] No passwords.
- [ ] No OAuth tokens.
- [ ] No private keys.
- [ ] No production credentials.
- [ ] No customer data.
- [ ] No unredacted PII.
- [ ] No sensitive screenshots.
- [ ] Example domains use `example.com` or `.test`.
- [ ] Example emails use `example.test`.
- [ ] Security-sensitive docs have owner.
- [ ] Security-sensitive docs link to Book III security docs.
